The author wrote Program 4.8 as a solution to the classic problem on page 104+, 4.4.3 The Sleeping Barber.  He noted that some of his information came from ([41], Problems 17, 18 on pages 72-73):  Andrew S. Tanenbaum, Modern Operating Systems, Prentice-Hall, 1992.

    In order to execute the author's basic program, one must go to the author's website and download the slba.java program.  It doesn't work, because it depends upon a number of unique supporting classes which the author has written.  To determine what is needed, one must compile the program and see what the error messages are.  Then one must create a directory called c:\LIB\UTILITIES\ and a second directory called c:\LIB\SYNCHRONIZATION.  The c:\LIB must be added to the path statement.  In c:\LIB\UTILITIES, one must download GetOpt.java, MyObject.java, and Scheduler.java.  In c:\LIB\SYNCHRONIZATION, one must download BinarySemaphore.java, ConditionVariable.java, CountingSemaphore.java, MessagePassing.java, MessagePassingRoot.java, NotImplementedMethodException.java, and Semaphore.java.  All of these will compile except Semaphore.java which is looking for WouldBlockException method.  I have not been able to find a file for this method.  I am not sure what the author is attempting to do with such a call.

    8 May 1999 Notes:  I learned from the author that there is a zip file that contains everything.  I have downloaded it and unzipped it.  I found the missing file.  Now the Java compiler can complete the whole task.  This exercise asks the student to execute the program.  I attempt to do that, but now the program won't execute.  Turns out the problem is that the author had a typo in the problem.  One must type "java SleepingBarber" instead of "java SleepingBarbers."  It now executes and yields similar results.
